# Автоматическое выделение лексикосемантических кластеров глаголов в русском языке с помощью алгоритма поиска сообществ в графе
* Данный репозиторий - перевыложенный проект для предмета "Математика для лингвистов", ФиКЛ ВШЭ, 2025/2026.
* Частично по этой причине в нескольких местах код избыточно усложнен, и по-хорошему его надо сильно дорабатывать.
* Файлы pr.ipynb и clusters.joblib - сам проект; остальные - ранние версии и эксперименты.
## Общая информация
* Целью проекта является автоматическое выделение глагольных лексических кластеров. Это может быть использовано для построения словарей синонимов и выделения семантических полей.
* Идея: берем вектора из модели дистрибутивной семантики, фильтруем глаголы; строим взвешенный граф, где каждый глагол присоединен к 10 своим ближайшим соседям; применяем алгоритм выделения сообществ.
* Весь код содержится в файле pr.ipynb; также в нем содержатся примеры выделенных кластеров. clusters.joblib - сохраненный результат запуска алгоритма; в pr.ipynb есть инструкция, как с ним работать.
## Особенности реализации
* Новшеством является алгоритм поиска ближайших соседей. В имплементации gensim [[1](#link1)], используемой для загрузки модели, ближайшие соседи ищутся через нахождение косинусных расстояний до всех словарных входов модели (https://github.com/piskvorky/gensim/blob/develop/gensim/models/keyedvectors.py - метод `KeyedVectors.most_similar`), сортировку и фильтрацию самых похожих; асимптотическая сложность поиска любого количества ближайших соседей для одного элемента в таком случае составляет $O(N)$ (где $N$ - количество входов модели). Существуют более эффективные способы нахождения ближайших соседей - поиск ближаших соседей с помощью Ball Tree (используемого в проекте) может иметь сложность $O(\log N)$. Однако, стоит отметить, что в высокоразмерном пространстве сложность, вероятно, оказывается выше. Более быстрые, но приближенные методы не используются, так как задача проекта - исследовательская, и использовать приближенные методы для нее нежелательно.
* При создании объекта помимо списка кластеров создается словарь с сохраненными хэшами кластеров. Это позволяет при нахождении кластера для объекта не перебирать все кластеры (линейный поиск, время $O(N)$), а доставать хэш кластера в одном словаре и кластер по хэшу в другом (что в среднем делается за время $O(1)$). Для хэширования использован алгоритм sha256 в имплементации библиотеки hashlib.
* Для поиска сообществ используется лувенский алгоритм. Его преимуществом является возможность учета веса ребер, что используется в проекте. Так как вершины перебираются в случайном порядке, результат может оказаться неодинаковым при разном их выборе; это нежелательно. Тем не менее, алгоритм намного быстрее главной альтернативы - алгоритма Гирван-Ньюмена.
* Вектора слов нормализуются до одинаковой длины, так как в дистрибутивной семантике их длина обычно считается нерелевантной для значения слов. Каждому ребру приписана евклидова дистанция между словами.
## Пояснения
* Можно было бы соединять ребрами не только ближайших соседей, а все со всеми; однако, это дало бы $C^{16000}_2>100,000,000$ (по примерному количеству правильно лемматизированных глаголов в выбранной модели) ребер, что представляет сложность для вычислений.
* Некоторые параметры алгоритма - resolution лувенского алгоритма, количество объектов в листе Ball Tree, количество соседей для записи в граф - были подобраны эмпирически; возможно, для более конкретных задач существуют более оптимальные их значения.
* Использована модель векторной семантики ruscorpora_upos_skipgram_300_5_2018 [[2](#link2)] (URL: https://rusvectores.org/ru/models/).
